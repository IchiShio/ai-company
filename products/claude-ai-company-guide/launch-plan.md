## 1. コンテンツ品質レビュー

### 総評

全7章+付録4つ、合計約3,900行。構成は論理的で、「はじめに」で興味を引き、第1章で設計思想、第2-3章で基礎（スキル+チェッカー）、第4章でパイプライン、第5章で部長、第6章で知識ベース、第7章でCOOと段階的にスケールアップする流れが非常によくできている。読者が「自分でもできそう」と感じる難易度設計になっている。

### 章間の整合性

- OK: 各章末の「次の章では〜」が正しく次章の内容を予告している
- OK: 第1章の「4原則」が後続章で一貫して適用されている
- OK: 「2回連続FAILルール」が第1章・第3章・第4章・第5章で一貫
- OK: 組織図（社長→COO→部長→社員）が全章で統一

### 重複箇所（軽微）

- 「2回連続FAILルール」の説明が第1章・第3章・第4章で繰り返されている。ただし各章で文脈が異なる（設計思想→チェッカー設計→パイプライン中断）ため、読者にとっては復習になり有益。**修正不要**と判断。
- メモリと知識ベースの違いテーブルが第6章と付録Bで重複。内容はほぼ同一。→ **軽微。付録Bは参照用なので許容範囲。**

**はじめに**: 非常に強い。「1人で会社を回している。社員は全員AI。」のフックは秀逸。SEOパイプライン・X投稿の具体例で「本当にできるんだ」と信じさせる構成。

## 2. note.com 販売戦略

### アカウント・ブランディング

**推奨案: @ichi_eigoのnoteアカウントで販売**

理由:
- 4万フォロワーの@ichi_eigoから誘導する前提なら、同一人物であることを隠す意味がない
- 新規匿名アカウントだと信頼性ゼロからスタートになり、3,980円のハードルが高すぎる
- @ichi_eigoの「英語学習×テック」のブランドとClaude Code活用は親和性が高い

**noteアカウント名候補（社長判断）:**

| 案 | メリット | デメリット |
|----|---------|-----------|
| A. @ichi_eigo（既存アカウント流用） | フォロワーからの信頼がそのまま使える | 英語学習コンテンツとAI系が混在する |
| B. 新規アカウント（例: @ai_company_guide） | コンテンツの一貫性 | ゼロからの信頼構築が必要 |
| C. @ichi_eigo で販売し、プロフィールに「AI×英語学習の自動化」を追加 | 既存信頼+テーマ拡張 | プロフィール変更の手間 |

**推奨: 案C。** @ichi_eigoアカウントで販売し、プロフィールに「Claude Codeで事業を自動化」的な一文を追加。英語学習運営の「裏側」を公開するスタンスは、フォロワーにとっても興味深いはず。

### 記事構成

**推奨: 1本の有料記事（マガジンではなく単品）**

理由:
- 章ごと分割は購入のハードルが上がり、トータル売上が下がるリスク
- マガジンは追加記事を書き続ける負担が発生する
- 単品3,980円なら「買い切り」の安心感がある
- note.comの有料記事は上限なく長文を書ける

**記事構成:**

```
[無料パート]
- はじめに（全文）
- 第1章: AI会社とは何か（全文）
- 第2章: 最初のスキルを作る（前半まで：SKILL.md作成〜呼び出しまで）

[有料パート ここから ¥3,980]
- 第2章 後半（スキル管理・5つのコツ）
- 第3章〜第7章（全文）

```

**有料ラインの設定ポイント:** 第2章の「やってみよう」セクション直後、「ただし、ここで1つ重大な問題があります。作ったスキルに仕事をさせてみると、AIは『やりました』と報告しておきながら、実は手を抜いていることがあるのです。」の直後で有料にする。この「問題提起→解決策は有料」の流れが最も購入率が高い。

### 価格設定

**3,980円は妥当。** 根拠:

- noteの技術系有料記事は 500〜5,000円が相場
- 3,900行・12ファイルのボリュームは書籍1冊分に匹敵
- 付録のテンプレート集（コピペで使える）は実用的価値が高い
- 競合がほぼ不在（後述）のため、値下げ圧力がない
- Claude Max $100/月を払っているユーザーにとって、$30程度は許容範囲

**ただし、ローンチ後の反応を見て調整する余地を残す:**
- 初週で10部未満 → 2,980円に値下げ検討
- 初週で30部以上 → 価格維持
- 100部超え → 4,980円に値上げ or 追加コンテンツ付きプレミアム版を検討

## 5. リスクと対策

### リスク1: 身元分離の漏洩

**リスク**: ガイド内の実例（native-real.com、@ichi_eigo）からcareermigaki関連が推測される

**現状評価**: **低リスク。** ガイド本文にcareermigaki / cm-* の言及は一切なし。native-real.com と @ichi_eigo のみ言及。この2つが同一人物であることは公開情報（Xプロフィール等）の範囲内であり問題ない。

**対策**:
- noteに掲載するスクリーンショット（もし追加する場合）にcareermigaki関連が映っていないことを確認
- noteアカウントのプロフィールにcareermigaki関連を記載しない
- 読者からの質問で「他にどんな事業を？」と聞かれた場合、native-real.com以外は言及しない

### リスク2: 価格への反応

**「高い」と言われるリスク:**
- 対策: 無料パートの充実度で「この品質の続きなら払う価値がある」と納得させる
- 対策: 付録テンプレート集の価値を強調（「コピペで使える」）
- 対策: 初週10部未満なら2,980円に値下げ

**「安い」と言われるリスク:**
- 対策: 追加コンテンツ（動画解説、個別相談付き等）を上位版として検討
- 対策: 100部超えたら値上げ

### リスク3: 競合出現

**現状**: 2026年3月時点で「Claude Codeでスキルを組織化する」という切り口のガイドは見当たらない。Claude Code自体がまだ新しいツールであり、先行者優位がある。

**対策**:
- 早く出す。完璧を求めすぎて遅れるのが最大のリスク
- 出した後も、運用実績を@ichi_eigoで継続的に発信し、「本家」としてのポジションを確立
- 購入者の声やフィードバックを取り入れてアップデートし、後発との差別化を維持

### リスク4: Claude Code仕様変更

**リスク**: Claude Codeのスキル仕様やAPI仕様が変更されると、ガイドの内容が陳腐化する

**対策**:
- ガイドの冒頭に「2026年3月時点の情報です」と明記
- 設計思想（1スキル=1責務、チェッカー、エスカレーション）は仕様変更に強い普遍的な考え方
- 大きな仕様変更があった場合は、note記事を更新（購入者は更新版も読める）

### リスク5: 技術的な質問への対応

**リスク**: 購入者から「動かない」「エラーが出る」等の技術質問が来る

**対策**:
- ガイド内の前提環境を明確にしている（Claude Max/Pro、Claude Code CLI、ターミナル基本操作）
- noteのコメント欄で「よくある質問」として回答を蓄積
- 個別のデバッグ対応は行わない方針（ガイドは「設計図」であり「サポート契約」ではない旨を明記）
