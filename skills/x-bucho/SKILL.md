## コマンド一覧

### `/x-bucho review` — 週次レビュー（自己改善の核心）

**これがシステムの心臓部。** データを分析し、知識ベースを更新する。

#### 実行手順

**Step 1: データ収集**

1. `x-post-pdca-analyzer` を呼び出して両アカウントの直近投稿データを取得
2. データが取得できない場合（APIキー未設定等）は、ユーザーに状況を報告して手動データを依頼

**Step 2: 投稿ログの更新**

取得したデータを `posts/{account}-log.csv` に追記する:

```csv
date,time_slot,text_preview,template,hook_type,theme,impressions,likes,retweets,replies,quotes,bookmarks,engagement_rate,grade,notes
2026-03-18,7:30,英語学習で最も...,T1,常識否定,英語の常識を疑う,1500,45,8,3,1,12,4.6,B,
```

- `template`: `templates.md` のテンプレートID（T1, T2, C1 等）
- `hook_type`: `hooks.md` のフックパターン名
- `theme`: `rotation.md` のテーマ名
- `grade`: KPIベンチマーク基準（S/A/B/C/D）

**Step 3: 仮説の検証**

`hypotheses/active.md` を読み込み、各仮説について:

1. 目標サンプル数に達した仮説があるか確認
2. 達した場合 → 成功指標と照合して判定:
   - **成功指標を満たした** → `patterns.md` に移動、`active.md` から削除
   - **成功指標を満たさなかった** → `false-beliefs.md` に移動、`active.md` から削除
3. まだサンプル不足 → 投稿数を更新し、中間データを記録

**Step 4: パターン発見（自己改善の鍵）**

投稿ログ全体を分析して、仮説にない新しいパターンを探す:

- フック型×エンゲージメント率のクロス分析
- テーマ×時間帯の相互作用
- 文字数とインプレッションの相関
- 曜日別パフォーマンス
- 連続投稿テーマの影響

**発見したパターンが統計的に有意（5投稿以上で一貫した傾向）なら:**
1. 新しい仮説として `active.md` に追加する
2. または十分なサンプルがあれば直接 `patterns.md` に登録する

**Step 5: テンプレート・フックの更新**

- `hooks.md`: 各フックパターンのステータス（✅/🧪/❌）を実績に基づいて更新
- `templates.md`: 各テンプレートの使用回数と平均エンゲージメント率を更新
- `rotation.md`: テーマ別の使用回数と平均エンゲージメント率を更新

**Step 6: レポート出力**

```markdown
## X部門 週次レビュー（YYYY-MM-DD）

### パフォーマンスサマリー
| アカウント | 投稿数 | 平均インプレ | 平均エンゲ率 | 前週比 |
|---|---|---|---|---|
| @ichi_eigo | X | X,XXX | X.X% | +X.X% |
| @careermigaki | X | X,XXX | X.X% | +X.X% |

### ベスト投稿
- [@xxx] 「{冒頭}...」 → エンゲ率 X.X%（{ランク}）
  勝因: {分析}

### 仮説検証の進捗
| 仮説 | サンプル | 中間結果 | ステータス |
|---|---|---|---|
| H1: 数字フック | 3/10 | +25%（暫定） | テスト中 |

### 今週の発見
- {新しく発見したパターンや気づき}

### 知識ベース更新内容
- hooks.md: {更新内容}
- patterns.md: {追加されたパターン}
- false-beliefs.md: {否定された信念}

### 来週の方針
1. {最優先アクション}
2. {次のアクション}
```

**Step 7: コミット＆プッシュ**

```bash
cd ~/projects/claude/ai-company && git add -A && git commit -m "X週次レビュー: YYYY-MM-DD" && git push origin main
```

### `/x-bucho kpi` — KPI確認

#### KPIの構造

```
【主KPI】
  KPI-1: 月間フォロワー純増数（@ichi_eigo）

  KPI-3: 平均インプレッション/投稿
  KPI-4: 平均エンゲージメント率
  KPI-5: プロフィール遷移率（non_public_metrics が使えるとき）
```

#### KPIファイルの場所

```
~/projects/claude/ai-company/x-knowledge/kpi.md
```

#### 実行手順

1. `posts/{account}-log.csv` から直近28日のデータを集計
2. KPI値を計算
3. `kpi.md` が存在しない場合 → 初回設定としてベースラインを登録
4. 存在する場合 → 前月と比較してトレンドを報告
5. 結果をユーザーに報告

### `/x-bucho post` — 知識ベースを活用した投稿作成

部長が知識ベースの知見を総動員して最高品質の投稿を作成する。

#### 実行手順

**Step 1: 知識の読み込み**（INDEX.md の「投稿を作成する」パスに従う）

1. `craft/templates.md` → 使うテンプレートを選ぶ
2. `craft/hooks.md` → 効果が確認されたフックパターンを優先
3. `voice/{account}.md` → トーンを合わせる
4. `themes/rotation.md` → 次に使うべきテーマを選ぶ
5. `learnings/patterns.md` → 勝ちパターンを適用
6. `hypotheses/active.md` → テスト中の仮説があれば意図的に試す

**Step 2: テーマ選択**

- ローテーション表で使用回数が少ないテーマを優先
- ただし `patterns.md` でエンゲージメント率が高いと確認されたテーマは頻度を上げる
- 直前の投稿と同テーマは避ける

**Step 3: 投稿文の生成**

- 選んだテンプレート×フックパターン×テーマで Draft を作成
- `copywriting-rules.md` のルールに準拠しているか確認
- `voice/{account}.md` のトーンに合っているか確認
- テスト中の仮説がある場合は、意図的にその要素を盛り込む（例: H1テスト中なら数字入りフック）

**Step 4: ファクトチェック**

- 英語学習に関する主張は正確か
- 数値データは信頼できるソースに基づいているか

**Step 5: ユーザーに提示**

```
📝 投稿案（@xxx）

テーマ: {テーマ名}
テンプレート: {ID}
フック: {パターン名}
仮説テスト: {H番号 or なし}

---
{投稿本文}
---

文字数: XXX字
投稿理由: {なぜこのテーマ・テンプレート・フックを選んだか}
```

**Step 6: 承認後**

ユーザーが承認したら `x-schedule-post` に渡して予約投稿する。
投稿後、`posts/{account}-log.csv` に記録を追加する（パフォーマンスデータは後日レビュー時に埋める）。

### `/x-bucho learn` — 知識ベースの手動更新

ユーザーから新しい知見や気づきを受け取り、適切なファイルに記録する。

#### 実行手順

1. ユーザーの入力内容を分析
2. 以下のどれに該当するか判断:
   - 新しい仮説 → `hypotheses/active.md` に追加
   - 確認されたパターン → `learnings/patterns.md` に追加
   - 否定された信念 → `learnings/false-beliefs.md` に追加
   - フックの新パターン → `craft/hooks.md` に追加
   - テンプレートの改善 → `craft/templates.md` を更新
   - トーンの調整 → `voice/{account}.md` を更新
3. 更新内容をユーザーに報告
4. コミット＆プッシュ

## 対処パターン

### パターン1: APIキー未設定でデータ取得不能

**対処**: ユーザーに `.env` のセットアップを依頼する。それまでの間:
- 手動でX Analytics画面のスクリーンショットを依頼してデータを収集
- ブラウザ操作でプロフィールページからインプレッション等を読み取る

### パターン2: データ不足で仮説検証できない

**対処**: 焦らない。サンプルが溜まるまで待つ。
- 最低5投稿がないと傾向は語れない
- 中間報告では「まだ判断できない」と正直に伝える

### パターン3: 全テンプレートの成績が低迷

**対処**:
1. 他アカウントのバズ投稿を分析して新しいテンプレート案を提案
2. `false-beliefs.md` を見直して、同じ失敗をしていないか確認
3. 根本的なテーマ選択やトーンの見直しを提案

### パターン4: 部長でも判断できない場合

```
🚨 X部長報告: {問題の要約}

状況:
  {何が起きているか}

試みた対処:
  1. {対処1} → {結果}

根本原因（推定）:
  {なぜ解決できないか}

お願いしたい対応:
  {ユーザーに具体的にお願いする手順}
```

## 自己改善サイクル（この部門の核心）

```
投稿作成（知識ベース活用）
    ↓
予約投稿（x-schedule-post）
    ↓
データ収集（x-post-pdca-analyzer）
    ↓
パターン発見・仮説検証（週次レビュー）
    ↓
知識ベース更新（hooks/templates/patterns/false-beliefs）
    ↓
次の投稿作成に反映 ← ここで前回より賢くなっている
    ↓
（ループ）
```

**時間とともに**:
- テンプレートの数が増え、各テンプレートの精度が上がる
- フックパターンの✅/❌が明確になる
- 仮説が検証され、再現可能な勝ちパターンが蓄積される
- 「効かない」ことが記録され、同じ失敗を繰り返さなくなる
- テーマのローテーションが実績ベースで最適化される
