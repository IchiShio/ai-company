# native-real.com SEO分析スキル

## 部門目標

**主目的: アフィリエイト収入の増加**（A8.net経由のオンライン英会話サービス申込）
検索順位の改善はその手段であり、目的ではない。

### サイトの収益構造

| ティア | URL パターン | 収益との関係 | affiliate_weight |
|---|---|---|---|
| Tier 1（収益直結） | `/`（ランキングTOP）・`/services/*/` | A8.netアフィリエイトリンク → 直接収益 | **3.0** |
| Tier 2（収益支援） | `/articles/*/` | 比較・ガイド記事 → 内部リンクでサービスページへ誘導 | **1.5** |
| Tier 2.5（エンゲージメント） | `/listening/` | ListenUpクイズ → axis別CTAでサービスページへ誘導 | **2.0** |
| Tier 3（SEOサポート） | `/real-phrases/*/` | トラフィック獲得・サイト権威性向上 | **1.0** |

**Top 5 アクション選定はこの収益ティアを最重要視する。**
同じクリック増でもTier 1ページへの施策を最優先する。

## 概要

このスキルは「分析者」ではなく「参謀」として振る舞う。数値の羅列ではなく、
「○○ページのtitleタグを△△に変更する」レベルの具体的なアクションを提案する。

対象サイト: https://native-real.com（英語学習サービス比較サイト）

データ収集は `native-real-data-collector` スキルが担当。このスキルは分析・提案のみを行う。

## CSVパース仕様

### Search Console CSV

ヘッダー列名（日本語）:
```
上位のクエリ / クリック数 / 表示回数 / CTR / 掲載順位
上位のページ / クリック数 / 表示回数 / CTR / 掲載順位
日付 / クリック数 / 表示回数 / CTR / 掲載順位
```

- **CTR列**: `"14.37%"` → float `14.37`（0〜100で管理、パーセント表記を数値化）
- **順位・CTRが空欄** → `null`（計算対象外）
- ファイル名が文字化けしている場合はヘッダー行で識別する

### GA4 CSV

- **`#` で始まる行を全てスキップ** → 最初の非コメント行がヘッダー
- **エンゲージメント率**: `0.315` → 表示時は `31.5%` に変換（0〜1の小数）
- **エンゲージメント時間**: 秒数（例: `27.2`）

### Ahrefs CSV

- **列名の注意**:
  - `best_position`（`position` ではない）
  - `sum_traffic`（`traffic` ではない）
  - `domain_rating_source`: 空の場合あり → `0` 扱い
- 列名マッチ失敗時は位置ベース（インデックス）で参照し、注記を出力

## Step 1: 7つの分析

### A. クイックウィン（queries.csv + pages.csv）

**対象**: 順位4〜20 AND インプレッション ≥ 5 のクエリ

**優先度スコア計算**:
```
position_weight:
  4〜10位  → 1.5
  11〜15位 → 1.0
  16〜20位 → 0.6

優先度スコア = インプレッション × position_weight
```

**出力形式**:
```
| クエリ | 順位 | インプレ | クリック | CTR | 優先度スコア |
（スコア降順、上位15件）
```

**狙い**: 少しの改善でランクアップ→クリック増が見込める「短期決戦」キーワード。

---

### B. 高インプレ低CTR（pages.csv）

**対象**: インプレッション ≥ 20 のページ

**期待CTR基準**:
```
1〜3位  → 15%
4〜6位  → 5%
7〜10位 → 2%
11〜20位 → 0.5%
```

**CTR不足度** = (期待CTR - 実CTR) / 期待CTR

- 0.8以上 → 「緊急」タグ付き
- 0.5以上 → 「要対応」

**出力形式**:
```
| ページ | 順位 | インプレ | 実CTR | 期待CTR | CTR不足度 | タグ |
（CTR不足度降順）
```

**狙い**: 既に検索には表示されているのにクリックされていないページ。
titleタグ・metaデスクリプション改善で即効性が高い。

---

### C. チャネル品質（ga4_traffic.csv）

**品質スコア計算**:
```
品質スコア = エンゲージメント率 × 0.4
           + (平均エンゲージメント時間 / 全チャネル最大時間) × 0.4
           + (セッション数 / 全チャネル最大セッション) × 0.2
```

**評価ランク**:
```
S: スコア ≥ 0.7
A: 0.5〜0.7
B: 0.3〜0.5
C: < 0.3
```

**出力形式**:
```
| チャネル | セッション | エンゲージ率 | 平均時間 | スコア | ランク |
（スコア降順）
```

---

### D. コンテンツ品質（ga4_pages.csv）

**分類ロジック**:
```
ユーザー ≥ 5 AND 平均エンゲージメント時間 ≥ 30秒 → 「スター」（伸ばすべき）
ユーザー ≥ 5 AND 平均エンゲージメント時間 < 15秒 → 「要改善」（冒頭で離脱）
ユーザー < 5 AND 平均エンゲージメント時間 ≥ 30秒 → 「潜在」（流入増で化ける）
その他 → 「低優先」（出力省略）
```

**出力形式**: スター / 要改善 / 潜在 のみ表示
```
### スター（伸ばすべきページ）
| ページ | ユーザー | 平均時間 |

### 要改善（冒頭改善で離脱率改善）
| ページ | ユーザー | 平均時間 |

### 潜在（流入施策で伸びる）
| ページ | ユーザー | 平均時間 |
```

---

### E. キーワード機会（ahrefs_keywords.csv）

**機会スコア計算**:
```
position_discount:
  best_position 11〜20位 → 1.0
  best_position 21〜50位 → 0.7
  best_position 51位以降 → 0.3

機会スコア = volume × position_discount
```

**集計**: 同一URLのキーワードはグループ化し、合計volumeとベストスコアを表示

**出力形式**:
```
| URL | キーワード（代表） | KW数 | 合計Vol | ベスト順位 | 機会スコア |
（スコア降順、上位10件）
```

---

### F. バックリンク品質（ahrefs_backlinks.csv）

**スパム判定**: アンカーテキストまたは参照ドメインに以下を含む場合
```
スパムキーワード: seo, backlink, rank, boost, dr, da, link building, cheap, buy
```

**評価**:
```
スパム率 = スパムリンク数 / 総リンク数
≥ 50% → Disavow推奨（緊急）
≥ 20% → 要監視
< 20% → 良好
```

**出力形式**:
```
総バックリンク: XX件 / 参照ドメイン: XX件
スパム率: XX%（評価）
DR別分布: DR0-10: XX件 / DR11-30: XX件 / DR31-60: XX件 / DR61+: XX件
```

---

### G. 28日間トレンド（dates.csv）

**計算**:
```
前半14日クリック合計 = 日付昇順で前半14日の sum(クリック数)
後半14日クリック合計 = 日付昇順で後半14日の sum(クリック数)
変化率 = (後半 - 前半) / 前半 × 100
```

**判定**:
```
> +100% → 「急成長」
> +20%  → 「成長」
±20%   → 「安定」
< -20% → 「減少」
< -50% → 「急落」
```

⚠️ **「急落」の場合**: 原因調査アクションを Top 5 の**1位**に無条件で挿入する。

**出力形式**:
```
前半14日: XXクリック / XXインプレ
後半14日: XXクリック / XXインプレ
変化率: +XX% [判定]
（折れ線グラフは省略、数値で表現）
```

### H. ListenUpエンゲージメント分析（ga4_pages.csv + GA4イベント）

**データソース**: ga4_pages.csv の `/listening/` 行 + GA4カスタムイベント（quiz_session_complete）

**基本指標**（ga4_pages.csv から取得）:
```
ListenUp PV数 / エンゲージメント率 / 平均エンゲージメント時間
```

**GA4イベントベースの分析**（データがある場合）:

| 指標 | 確認方法 | 判定 |
|---|---|---|
| モード別セッション数 | `quiz_session_complete` の `quiz_mode` 集計 | daily/weak/revengeが合計20%未満 → モード認知不足 |
| axis別正解率 | `axis_speed_acc`〜`axis_distractor_acc` の平均 | 特定axisが他より20pt以上低い → 問題品質要確認 |
| 完了率 | セッション開始数 vs `quiz_session_complete` 数 | 50%未満 → 離脱原因調査 |

**出力形式**:
```
### H. ListenUpエンゲージメント
- PV: XX（サイト内順位: X位）
- エンゲージメント率: XX%（平均XX秒）
- モード分布: normal XX% / daily XX% / weak XX% / revenge XX%
- axis正解率: speed XX% / reduction XX% / vocab XX% / context XX% / distractor XX%
- 判定: [良好/要改善]（理由）
```

**注意**: GA4イベントデータはga4_pages.csvには含まれない。GA4管理画面のイベントレポートから取得可能な場合のみ詳細分析を行う。ga4_pages.csvの基本指標だけでも「スター/要改善/潜在」分類に反映する。

### I. 競合コンテンツギャップ分析（ahrefs_competitor_keywords.csv）

**目的**: ベンチマーク競合がランクインしていて、native-real.comがカバーしていないキーワード・トピックを特定する。

**データソース**:
- `ahrefs_competitor_keywords.csv`（競合の有機キーワード）
- `ahrefs_keywords.csv`（自サイトの有機キーワード）
- `site-context.md`（競合情報・ターゲットキーワード）

**分析手順**:

1. **キーワードギャップの抽出**:
   ```
   競合キーワードのうち、自サイトの ahrefs_keywords.csv に存在しないもの
   → ギャップキーワード
   ```

2. **ギャップスコア計算**:
   ```
   relevance_weight（英語学習・英会話関連度）:
     「英会話」「英語」「オンライン英会話」等を含む → 2.0
     それ以外                                       → 1.0

   gap_score = volume × relevance_weight
   ```

3. **カテゴリ分類**:
   - **即対応可能**: 既存ページのtitle/コンテンツ調整でカバーできるギャップ
   - **新規コンテンツ候補**: 既存ページでカバーできないトピック（※社長方針で後回し）
   - **対象外**: native-real.comのスコープ外（競合の自社サービス名等）

**出力形式**:
```
### I. 競合コンテンツギャップ

ベンチマーク: [競合名]（[ドメイン]）

#### 即対応可能なギャップ
| キーワード | 競合順位 | 月間Vol | 対応既存ページ | 推奨アクション |
（gap_score降順、上位10件）

#### 新規コンテンツ候補（参考・既存改善完了後に検討）
| キーワード | 競合順位 | 月間Vol | トピック案 |
（上位5件）

#### 競合との重複キーワード比較
| キーワード | 自サイト順位 | 競合順位 | Vol | 自サイト有利/不利 |
（自サイトより競合の方が上位のキーワード、上位5件）
```

**狙い**: 「競合がランクインしていて自サイトにないキーワード」は、検索需要が実証済み。
特に「即対応可能」カテゴリの施策は、タイトル変更やセクション追加だけで獲得できる可能性がある。

### J. カバレッジ分析（coverage_errors.csv / coverage_chart.csv）

**目的**: Googleが直接指摘するインデックス問題（404、クロール済み未登録等）を検知し、即座に対処する。
これらはGoogleが「修正してください」と明示している問題であり、放置するとクロールバジェット浪費・サイト評価低下に直結する。

**データソース**:
- `coverage_errors.csv`（重大な問題 — 404、noindex等）
- `coverage_warnings.csv`（軽微な問題）
- `coverage_chart.csv`（日別の登録済み/未登録推移）

**分析手順**:

1. **重大な問題の分類**:
   ```
   「見つかりませんでした（404）」 → 緊急対応（301リダイレクトまたはsitemap除外）
   「noindex タグによって除外」    → 意図的か確認（privacy等はOK）
   「クロール済み - インデックス未登録」 → コンテンツ薄い可能性、リライト候補
   「検出 - インデックス未登録」    → クロールキュー待ち（様子見）
   ```

2. **インデックス率の計算**:
   ```
   インデックス率 = 登録済み / (登録済み + 未登録) × 100
   ```

3. **トレンド判定**（coverage_chart.csv）:
   ```
   直近7日間で登録済みが増加 → 「順調」
   直近7日間で未登録が増加  → 「要注意」（新ページ追加時は一時的に増えるため除外）
   ```

**出力形式**:
```
### J. カバレッジ（インデックス状況）

インデックス率: XX%（登録済みXX / 全XXページ）
トレンド: [順調/要注意/安定]

#### 重大な問題
| 問題 | ページ数 | 緊急度 | 推奨対応 |
（問題がない場合は「なし ✅」）

#### インデックス推移（直近7日）
| 日付 | 登録済み | 未登録 | 表示回数 |
```

**Top 5 への影響**:
- 404が1件以上ある場合 → 修正アクションをTop 5に**優先挿入**する（フェーズ: 基盤）
- 「クロール済み・インデックス未登録」が10件以上 → リライト候補としてTop 5検討対象に入れる

## Step 2: Top 5 優先アクションの生成

### 社長方針（2026-03-15）: 既存改善 > 新規作成

現在230ページあるがCTR 0%が大半。**新規記事作成はTop 5に原則入れない。**
既存ページのtitle改善・コンテンツリライト・CTA最適化・内部リンク整備を優先する。
新規作成はTop 5全枠を既存改善で埋められない場合のみ検討可。

### インパクト推定（月間クリック増）

| 施策タイプ | 計算式 | 工数 |
|-----------|-------|------|
| タイトル/メタ改善 | `インプレ × (期待CTR - 実CTR) / 100` | 小（30分以内） |
| コンテンツリライト | `Ahrefsボリューム × 目標順位CTR` | 中（2〜4時間） |
| 新規コンテンツ作成 | `ボリューム × 0.03` | 大（4時間以上） |
| アフィリエイトCTA最適化 | 定性（既存流入のCV率改善） | 小 |
| トレンド急落の調査 | 定性（順位回復の機会損失防止） | 小 |

### ランキングロジック

```
effort_weight:
  小 → 3.0
  中 → 1.0
  大 → 0.5

affiliate_weight（URL収益ティアに基づく）:
  Tier 1 = /（ルート）または /services/ を含む → 3.0
  Tier 2 = /articles/ を含む             → 1.5
  Tier 2.5 = /listening/                    → 2.0
  Tier 3 = /real-phrases/ 等                → 1.0

優先度スコア = インパクト × effort_weight × affiliate_weight
```

**制約**:
- 「工数: 大」は最大2件まで
- 同一ページへの施策は最も効果的な1件のみ
- 「急落」がある場合は1位に原因調査を強制挿入
- Tier 1ページに施策できる場合は積極的に選ぶ（スコアが低くても）

### 出力形式

```markdown
## Top 5 優先アクション

| # | フェーズ | アクション（具体的に） | 推定クリック増/月 | 収益ティア | 工数 | 根拠 |
|---|---------|----------------------|-----------------|-----------|------|------|
| 1 | 基盤 | [ページ名]のtitleタグを「[現在]」→「[提案]」に変更 | +XX回 | Tier 1 | 小 | インプレXX/CTRXX% |
| 2 | 基盤 | [ページ名]の冒頭2段落をリライト（離脱率改善） | +XX回 | Tier 2 | 小 | GA4: XX秒で離脱 |
| 3 | 最適化 | ... | ... | ... | ... | ... |
| 4 | 継続 | ... | ... | ... | ... | ... |
| 5 | コンテンツ | ... | ... | ... | ... | ... |
```

### フェーズタグの定義

各アクションに以下の4フェーズのいずれかをタグ付けする。
フェーズはアクションの性質を示すもので、実行順序はスコア順を維持する。

| フェーズ | 内容 | 例 |
|---------|------|---|
| **基盤** | title/meta/OGP等のメタデータ修正。即効性が高く工数が小さい | title変更、meta description改善 |
| **最適化** | 既存コンテンツの品質向上。CTR・エンゲージメント改善 | リライト、CTA改善、内部リンク追加 |
| **コンテンツ** | 新規ページ作成やコンテンツの大幅追加 | 新規記事、セクション追加 |
| **継続** | 定期的に繰り返す施策 | Disavow更新、問題追加、トレンド監視 |

**フェーズバランスの目安**: 1回のTop 5に「基盤」が2件以上含まれることが望ましい。
基盤施策がなくなった段階で「最適化」フェーズに移行していると判断する。

アクションは「誰が読んでもすぐ実行できる」具体性で書く。
「改善する」ではなく「○○を△△に変更する」「□□セクションを追加する」レベル。

## エッジケース処理

| ケース | 対処 |
|--------|------|
| 必須ファイル（queries/pages）不在 | 「データ未収集。先に `/native-real-data-collector` を実行してください」で終了 |
| GA4ファイルのみ欠落 | C/D分析をスキップ（「GA4データなし → C/D分析スキップ」と注記） |
| Ahrefsファイルのみ欠落 | E/F分析をスキップ |
| クエリ件数 < 10件 | 「データ蓄積中」フラグを立て、閾値を緩和（インプレ ≥ 5 → ≥ 2） |
| Ahrefs列名マッチ失敗 | インデックス（位置）ベースで参照し「列名フォールバック使用」と注記 |
| CTR列が "14.37%" 形式 | `%` を除去して float 変換（0〜100スケール維持） |
| GA4の `#` コメント行 | 全スキップ、最初の非コメント行をヘッダーとして解析 |

## Section K: kioku-shinai UX分析（追加セクション）

kioku-shinai（記憶しない英単語）はnative-real部門配下の新サービス。
data-collectorが収集した `kioku-shinai-feedback-YYYY-MM-DD.json` を読み込んで分析する。

### データファイル

```
{日付フォルダ}/kioku-shinai-feedback-YYYY-MM-DD.json
```

ファイルが存在しない場合はこのセクションをスキップする。

### 分析項目

#### K-1. 完走率・離脱分析

フィードバックが存在する = 完走したユーザー。
GA4が設定されるまでは完走数のみ計測（離脱率はGA4設定後に追加）。

```
完走数: {フィードバックレコード数}
```

#### K-2. フィードバック集計

各質問の回答分布を集計する:

```
理解度:
  0（まだよくわからない）: N件 (XX%)
  1（少し理解できた）:     N件 (XX%)
  2（かなり理解できた）:   N件 (XX%)
  3（完全に腑に落ちた）:   N件 (XX%)
  → 理解スコア（2+3の割合）: XX%

従来比較:
  0（覚えにくい）:       N件 (XX%)
  1（変わらない）:       N件 (XX%)
  2（やや覚えやすい）:   N件 (XX%)
  3（はるかに覚えやすい）: N件 (XX%)
  → 優位性スコア（2+3の割合）: XX%

量:
  0（多すぎる）:     N件 (XX%)
  1（ちょうどいい）: N件 (XX%)
  2（もっと欲しい）: N件 (XX%)

良かった要素（複数選択）:
  語源の分解:     N件
  認知言語学:     N件
  和訳のズレ:     N件
  記憶フック:     N件
  音声:          N件
  出題の多様さ:   N件
```

#### K-3. 正解率・レベル分析

フィードバックに `correct` / `total` が含まれている場合:
```
平均正解率: XX%
レベル別正解率: beginner XX% / intermediate XX%
```

#### K-4. 改善アクション提案

分析結果に基づき、kioku-shinai向けの具体的な改善アクションを最大3つ提案する。

**判断基準**:
- 理解スコア < 70% → 解説の質を改善（語源説明をより分かりやすく）
- 優位性スコア < 60% → 従来単語帳との差別化を強化
- 「多すぎる」が30%以上 → 解説量を削減、段階的表示に変更
- 「もっと欲しい」が50%以上 → 解説セクションを追加
- 正解率 > 90% → 難易度を上げる / 上位形式を増やす
- 正解率 < 40% → 難易度を下げる / ヒントを充実
