# native-real.com SEO分析スキル

**主目的: アフィリエイト収入の増加**（A8.net経由のオンライン英会話サービス申込）
検索順位の改善はその手段。「数値の羅列」ではなく「具体的なアクション提案」に徹する。
データ収集は `native-real-data-collector` スキルが担当。このスキルは分析・提案のみ行う。

---

## 収益ティア定義（affiliate_weight）

| ティア | URL パターン | 収益との関係 | weight |
|---|---|---|---|
| Tier 1（収益直結） | `/ranking/`・`/services/*/` | A8.net アフィリエイトリンク → 直接収益 | **3.0** |
| Tier 2（収益支援） | `/articles/*/` | 比較・ガイド記事 → 内部リンクでサービスページへ誘導 | **1.5** |
| Tier 2.5（エンゲージメント） | `/`・`/grammar/`・`/words/`・`/speaking/`・`/kioku-shinai/` | ツールページ → axis別CTAでサービスページへ誘導 | **2.0** |
| Tier 3（SEOサポート） | `/real-phrases/*/`・`/prompts/*/` | トラフィック獲得・権威性向上 | **1.0** |

※ `/` はListenUp（集客メイン）、`/ranking/` がサービス比較ランキング（Tier 1収益ページ）。

### 4. CSVパース仕様

**Search Console CSV（日本語ヘッダー）**:
- CTR列: `"14.37%"` → float `14.37`（0〜100スケール）
- 順位・CTRが空欄 → `null`（計算対象外）

**GA4 CSV**:
- `#` で始まる行を全スキップ → 最初の非コメント行がヘッダー
- エンゲージメント率: `0.315` → 表示時は `31.5%`（0〜1の小数）

**Ahrefs CSV**:
- カラム名注意: `best_position`（`position` ではない）、`sum_traffic`（`traffic` ではない）
- 列名マッチ失敗時 → インデックスで参照し注記を出力

## Step 1: 分析（A〜K）

### A. クイックウィン（queries.csv）

**対象**: 順位4〜20 かつ インプレッション ≥ 5 のクエリ

```
position_weight: 4〜10位 → 1.5 / 11〜15位 → 1.0 / 16〜20位 → 0.6
優先度スコア = インプレッション × position_weight
```

出力: `| クエリ | 順位 | インプレ | クリック | CTR | 優先度スコア |`（スコア降順、上位15件）

---

### B. 高インプレ低CTR（pages.csv）

**対象**: インプレッション ≥ 20 のページ

```
期待CTR: 1〜3位 → 15% / 4〜6位 → 5% / 7〜10位 → 2% / 11〜20位 → 0.5%
CTR不足度 = (期待CTR - 実CTR) / 期待CTR
0.8以上 → 「緊急」 / 0.5以上 → 「要対応」
```

出力: `| ページ | 順位 | インプレ | 実CTR | 期待CTR | CTR不足度 | タグ |`（CTR不足度降順）

---

### C. チャネル品質（ga4_traffic.csv）

```
品質スコア = エンゲージメント率 × 0.4
           + (平均エンゲージ時間 / 最大時間) × 0.4
           + (セッション数 / 最大セッション) × 0.2
ランク: S(≥0.7) / A(0.5〜0.7) / B(0.3〜0.5) / C(<0.3)
```

出力: `| チャネル | セッション | エンゲージ率 | 平均時間 | スコア | ランク |`（スコア降順）

---

### D. コンテンツ品質（ga4_pages.csv）

```
ユーザー ≥ 5 AND 平均エンゲージ時間 ≥ 30秒 → 「スター」
ユーザー ≥ 5 AND 平均エンゲージ時間 < 15秒 → 「要改善」
ユーザー < 5 AND 平均エンゲージ時間 ≥ 30秒 → 「潜在」
その他 → 出力省略
```

出力: スター / 要改善 / 潜在 の3セクションで `| ページ | ユーザー | 平均時間 |`

---

### E. キーワード機会（ahrefs_keywords.csv）

```
position_discount: 11〜20位 → 1.0 / 21〜50位 → 0.7 / 51位以降 → 0.3
機会スコア = volume × position_discount
```

同一URLのキーワードをグループ化。出力: `| URL | キーワード（代表） | KW数 | 合計Vol | ベスト順位 | 機会スコア |`（上位10件）

---

### F. バックリンク品質（ahrefs_backlinks.csv）

スパム判定（アンカーテキストまたは参照ドメインに含む場合）:
`seo, backlink, rank, boost, dr, da, link building, cheap, buy`

```
スパム率 ≥ 50% → Disavow推奨（緊急）
スパム率 ≥ 20% → 要監視
スパム率 < 20% → 良好
```

出力: `総BL: XX件 / 参照ドメイン: XX件 / スパム率: XX%（評価） / DR別分布`

---

### G. 28日間トレンド（dates.csv）

```
前半14日 vs 後半14日クリック合計で変化率を計算
> +100% → 急成長 / > +20% → 成長 / ±20% → 安定 / < -20% → 減少 / < -50% → 急落
```

⚠️ **「急落」の場合**: 原因調査アクションをTop 5の**1位に無条件で挿入**する。

---

### H. ListenUpエンゲージメント（ga4_pages.csv）

**データソース**: ga4_pages.csv の `/`（ルートページ）行を参照（ListenUpはトップページに移動済み）

```
PV数 / エンゲージメント率 / 平均エンゲージメント時間 を取得
```

GA4カスタムイベント（`quiz_session_complete`）がある場合のみ詳細分析:
- モード別分布（normal/daily/weak/revenge）
- axis別正解率（speed/reduction/vocab/context/distractor）
- 完了率 < 50% → 離脱原因調査を提案

---

### I. 競合コンテンツギャップ（ahrefs_competitor_keywords.csv）

ベンチマーク競合（site-context.md より）がランクインしていて自サイトにないキーワードを抽出。

```
relevance_weight: 「英会話」「英語」「オンライン英会話」含む → 2.0 / その他 → 1.0
gap_score = volume × relevance_weight
```

出力:
- 即対応可能ギャップ（既存ページのtitle/コンテンツ調整でカバー可）: 上位10件
- 競合との重複KW比較（競合が上位のもの）: 上位5件

---

### J. カバレッジ分析（coverage_errors.csv / coverage_chart.csv）

問題分類:
```
「見つかりませんでした（404）」 → 緊急（301リダイレクトまたはsitemap除外）
「noindex タグによって除外」    → 意図的か確認（privacy等はOK）
「クロール済み - インデックス未登録」 → コンテンツ薄い可能性、リライト候補
「検出 - インデックス未登録」   → クロールキュー待ち（様子見）
```

インデックス率 = 登録済み / (登録済み + 未登録) × 100

**Top 5への影響**:
- 404が1件以上 → 修正アクションをTop 5に優先挿入
- 「クロール済み・インデックス未登録」が10件以上 → リライト候補をTop 5検討対象に追加

---

### K. kioku-shinai UX分析（kioku-shinai-feedback-YYYY-MM-DD.json）

ファイルが存在しない場合はスキップ。

集計項目:
- 理解スコア（理解度2+3の割合） / 優位性スコア（従来比2+3の割合）
- 量フィードバック分布 / 良かった要素ランキング
- 平均正解率（`correct`/`total`がある場合）

改善判断基準:
```
理解スコア < 70%   → 語源説明をより分かりやすく
優位性スコア < 60% → 従来単語帳との差別化を強化
「多すぎる」≥ 30% → 解説量を削減
「もっと欲しい」≥ 50% → 解説セクションを追加
正解率 > 90%       → 難易度を上げる
正解率 < 40%       → 難易度を下げる / ヒントを充実
```

## Step 2: Top 5 優先アクション生成

### 方針: 既存改善 > 新規作成

現在230ページあるがCTR 0%が大半。**新規記事作成はTop 5に原則入れない。**
既存ページのtitle改善・コンテンツリライト・CTA最適化・内部リンク整備を優先する。

### インパクト計算

| 施策タイプ | 計算式 | 工数 |
|-----------|--------|------|
| タイトル/メタ改善 | `インプレ × (期待CTR - 実CTR) / 100` | 小（30分以内） |
| コンテンツリライト | `Ahrefsボリューム × 目標順位CTR` | 中（2〜4時間） |
| 新規コンテンツ作成 | `ボリューム × 0.03` | 大（4時間以上） |
| CTA最適化 | 定性（既存流入のCV率改善） | 小 |
| 急落原因調査 | 定性（順位回復の機会損失防止） | 小 |

### ランキングロジック

```
effort_weight: 小 → 3.0 / 中 → 1.0 / 大 → 0.5
affiliate_weight: Tier 1 → 3.0 / Tier 2 → 1.5 / Tier 2.5 → 2.0 / Tier 3 → 1.0

優先度スコア = インパクト × effort_weight × affiliate_weight
```

制約:
- 「工数: 大」は最大2件まで
- 同一ページへの施策は最も効果的な1件のみ
- 急落がある場合は1位に原因調査を強制挿入
- Tier 1（`/ranking/`・`/services/*/`）への施策がある場合は積極的に選ぶ

### 出力形式

```markdown
## Top 5 優先アクション

| # | フェーズ | アクション（具体的に） | 推定クリック増/月 | 収益ティア | 工数 | 根拠 |
|---|---------|----------------------|-----------------|-----------|------|------|
| 1 | 基盤 | /ranking/ のtitleを「[現在]」→「[提案]」に変更 | +XX回 | Tier 1 | 小 | インプレXX/CTR XX% |
```

### フェーズタグ

| フェーズ | 内容 |
|---------|------|
| **基盤** | title/meta/OGP修正。即効性が高く工数が小さい |
| **最適化** | 既存コンテンツの品質向上。CTR・エンゲージメント改善 |
| **コンテンツ** | 新規ページ作成やコンテンツの大幅追加 |
| **継続** | 定期的に繰り返す施策 |

1回のTop 5に「基盤」が2件以上含まれることが望ましい。
