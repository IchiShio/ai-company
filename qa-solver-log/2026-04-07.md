### g436 詳細

**修正前:**
```
stem: "We didn't find ___ useful information in the report."
answer: "any"
choices: ["any", "some", "much", "many", "no"]
```

**問題点:**
`"We didn't find much useful information"` は文法的に正しく自然な英語。
`much` + 不可算名詞（information）は否定文で標準的な用法（Cambridge Grammar 準拠）。
`any`（ゼロ量）と `much`（少量）が両方成立し、単一正解問題として機能しない。

**修正後:**
```
choices: ["any", "some", "few", "many", "no"]
expl: "否定文では any を使います。not ... any で「まったく～ない」を表します。few は可算名詞複数形に使い、不可算名詞（information）には使えません。"
```

`few` は可算名詞専用のため `information`（不可算）に使えず、明確な誤答として機能する。
**即修正・git commit 済み。**

### q1419 詳細
```
text: "He just nodded and walked away. Didn't say a word."
answer: "相手の反応が冷淡で会話が終わったことへの戸惑い"
```
"nodded and walked away without a word" は文脈次第で「冷淡な無視」にも「無言の承認」にも読める。
ただし "just nodded" + "didn't say a word" の組み合わせは英語話者の感覚では dismissive のニュアンスが強い。
answer は defensible だが、distractors（特に「深く納得して去ったと喜んでいる」）は正解と紙一重に見える。

### q3499 詳細
```
word: attribute A to B
answer: "AをBの原因・結果と見なす"
```
`attribute A to B` の正確な意味は「AをBが原因で生じたものと見なす（BがAの原因）」。
「原因・結果」という表記は A と B の役割が曖昧になる。
より精確には「AはBによるものと見なす」。間違いではないが、学習者が誤解する可能性がある。
