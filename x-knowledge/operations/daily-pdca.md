# @ichi_eigo 日次PDCAサイクル

## 基本設計

**1日4投稿 × 前日23時にセット → 毎日繰り返す**

投稿枠: 7:00 / 12:00 / 18:00 / 21:00 JST（GitHub Actions自動投稿）

---

## 毎晩23時の作業手順

### Step 1: 当日データ確認

`ichi-eigo-log.csv` の当日4投稿を確認。

| 枠 | データ成熟度（23時時点） | 判断精度 |
|---|---|---|
| 7:00投稿 | 16時間経過 | 高（ほぼ確定値） |
| 12:00投稿 | 11時間経過 | 高 |
| 18:00投稿 | 5時間経過 | 中（翌朝で確定） |
| 21:00投稿 | 2時間経過 | 低（参考値のみ） |

→ 7:00・12:00の結果を主判断材料にする。18:00・21:00は翌日確定値で検証。

### Step 2: パターン判定

各投稿をランク評価（S/A/B/C/D）：
- S: 10,000+ / A: 5,000+ / B: 3,000+ / C: 1,000+ / D: 1,000未満

当日S・A投稿 → そのフック型・テーマ・Factの組み合わせを翌日の参考にする
当日C・D投稿 → 原因特定（フック弱? テーマ再投稿? 曜日?）

### Step 3: 翌日4枠の設計

**チェックリスト（各投稿前に必ず確認）:**
- [ ] 同テーマ・同Factの直近投稿が7日以内にないか（P12違反チェック）
- [ ] フックは「強」または「中」か（FB5: 条件型・共感型は禁止）
- [ ] Factストック（`facts/`）に根拠があるか（創作禁止）
- [ ] 仮説テスト枠（H3〜H8）は今日消化すべきものがあるか

**4枠の配分目標:**
- 安定打（P9/P10/P11確認済み）: 2〜3枠
- 仮説テスト（H番号付き）: 1〜2枠
- 未検証フック型テスト（対比型・断言型・疑問提起型等）: 0〜1枠

**テーマローテーション:**
- 同テーマは7日以上間隔を空ける（P12）
- 1日4枠で同テーマを複数使わない
- 朝・昼は高インプ重視（P9/P11）、夜は実験枠に使える

### Step 4: schedule.jsonを更新してpush

```bash
cd ~/projects/claude/ai-company
# schedule.jsonを編集（翌日の4投稿を追加）
git add x-knowledge/posts/schedule.json
git commit -m "翌日投稿セット: YYYY-MM-DD"
git push origin main
```

## 週次レビュー（日曜夜 or 月曜朝）

日次PDCAで取りきれない学びを週次でまとめる：
- 7日分データが揃ったH番号仮説を判定（active.md → patterns.md or false-beliefs.md）
- patterns.mdに新しい勝ちパターンを追記
- false-beliefs.mdに否定されたパターンを追記
- 翌週の仮説テスト枠の計画
